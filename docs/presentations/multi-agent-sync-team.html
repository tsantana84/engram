Daily Workflow

Nothing changes for you

The sync layer is invisible. Here's what's happening under the hood every session.

1

Hook fires → observation stored in SQLite

Exactly as before. Zero latency. Your local DB is the source of truth.

2

Observation enqueued in sync_queue

A lightweight row is inserted alongside the observation. Async, non-blocking.

3

SyncWorker batches and pushes every 30s

Up to 100 items per request. Server deduplicates by content hash. If server is unreachable, items stay pending and retry next cycle.

4

Search merges local + team results

When you ask your agent about anything, it queries local SQLite AND the team server (3s timeout). Results are merged, deduped (local wins), and attributed by author.

⚠️

Offline? No problem. Queue fills locally and drains automatically when connectivity returns. Your local agent is fully unaffected.

Failure handling

ScenarioBehaviour
Network timeout / 5xxItem stays pending, retried next cycle. Increments attempt counter.
5 failed attemptsMarked failed, logged, skipped. Run engram sync retry to resume.
Server 4xx (bad key, conflict)Marked failed immediately. No retry. Check your API key.
Duplicate contentServer silently ignores. ON CONFLICT DO NOTHING. Always safe to resend.
Server query timeout (>3s)Search returns local results only. No error shown to agent.

Privacy

Keep sensitive content local

Default: everything you work on is shared. To exclude something, wrap it in a <private> tag. Stripping happens at the hook layer — the content never reaches your local DB or the server.

In any Claude Code message or tool output
// Everything outside the tag → shared normally

  • Bug discoveries and fixes
  • Architecture decisions
  • Code patterns and gotchas
  • Session summaries
  • File paths and component names

🚫 Stays local (stripped at hook)

  • Credentials and API keys
  • Customer PII or sensitive data
  • Internal URLs or IPs
  • Anything in <private>…</private>

Sync commands

Day-to-day sync management. All under the engram sync subcommand.

$ engram sync pause
⏸ Sync paused. Queue filling locally.

Configuration

All sync settings

All settings go in ~/.claude-mem/settings.json. Defaults work for most teams.

Setting Default Purpose
CLAUDE_MEM_SYNC_ENABLED false Master toggle. Set to true to enable sync.
CLAUDE_MEM_SYNC_SERVER_URL null URL of your central server. e.g. https://engram.your-team.com
CLAUDE_MEM_SYNC_API_KEY null Your personal API key from the admin. Starts with cmem_ak_
CLAUDE_MEM_SYNC_AGENT_NAME null Your display name on team search results. Must match registered name.
CLAUDE_MEM_SYNC_INTERVAL_MS 30000 Push interval in milliseconds. Default: 30 seconds.
CLAUDE_MEM_SYNC_TIMEOUT_MS 3000 Server query timeout. Falls back to local-only if exceeded.
CLAUDE_MEM_SYNC_MAX_RETRIES 5 Max attempts before marking an item failed. Run sync retry to reset.
