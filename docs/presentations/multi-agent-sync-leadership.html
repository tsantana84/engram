Architecture

How it works

A sync layer sits on top of each local engram instance. No local changes — your SQLite database stays the source of truth. The server just collects and serves what's been shared.

1

Agent observes & stores locally

Claude Code hooks capture what you worked on — decisions, bugs fixed, discoveries. Stored instantly in your local SQLite database. Always works offline.

2

Sync queue fills asynchronously

Each new observation is added to a lightweight sync_queue table. No blocking. No latency on your local experience.

3

Background worker pushes in batches

Every 30 seconds, up to 100 pending items are pushed to the central server via HTTPS. Idempotent — safe to retry. Fails gracefully if the server is down.

4

Team knowledge is immediately searchable

Searches automatically merge local results with team results. Each observation is attributed to its author. Local copy always wins on deduplication.

Benefits

What the team gains

🧠

Institutional memory that persists

Architectural decisions, debugging breakthroughs, and lessons learned accumulate in a shared corpus that outlasts any single engineer.

⚡

Zero impact on local experience

Local SQLite stays the source of truth. Sync is async, offline-safe, and completely invisible to the agent during normal use.

🔍

Team-wide context in every search

Ask your agent about any codebase topic and get insights from the full team — not just your own sessions.

🏷️

Attribution & provenance

Every observation is tagged with its author and source. You always know whose experience produced an insight.

🛡️

Privacy by default

Sensitive content stays local with a simple <private> tag. No configuration needed — the tag system already exists.

🐳

Simple self-hosted deployment

A single Docker Compose file spins up the Postgres server and the engram central service. No cloud vendor lock-in.

Privacy Model

Default share, opt-out via tags

The team gets maximum knowledge by default. Engineers opt specific content out of sharing using a tag that already exists in engram today.

🔒

How it works

Tag any content with <private> and it is stripped at the hook layer — before data ever reaches the local database. It never enters the sync queue and never touches the server.

I found the issue <private>customer API key was abc123xyz — rotate immediately</private> in the config file.
✅

What's shared

Code discoveries, debugging sessions, architecture decisions, patterns, and learnings — the knowledge your team needs.

🚫

What stays local

Anything wrapped in <private> tags: credentials, sensitive customer data, personal notes, anything you choose.

🔑

Authentication

Each team member has a unique API key (cmem_ak_…) registered with the server. Keys are bcrypt-hashed. No OAuth complexity.
