Memory Quality

ConflictDetector

Stale observations pollute the shared brain. ConflictDetector classifies every outgoing observation before it reaches Supabase.

⚠️
The problem: An observation written while debugging a wrong hypothesis can mislead every agent on the team. Unmerged-branch observations are even worse — they may contradict main.

Pipeline (when CLAUDE_MEM_PROVIDER is set)

  1. Fetch top-5 similar obs from Supabase
  2. Pass to LLM with structured classification prompt
  3. Receive action → execute

Four actions

ADD New info — store normally
UPDATE Supersedes existing — invalidate old
INVALIDATE Old obs appears wrong — remove it
NOOP Duplicate — drop silently

Provenance columns

ColumnTypePurpose
git_branchTEXTBranch at write time — flags unmerged observations
invalidated_atINTEGEREpoch when superseded (NULL = still valid)
validation_statusTEXTunvalidated / validated / invalidated
✓
Safe default: CLAUDE_MEM_PROVIDER not set → ConflictDetector disabled, all observations pass as ADD. Set it in ~/.engram/settings.json to enable classification.
