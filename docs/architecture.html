Subsystems

Hooks
▼
  • SessionStartinjects past context into the prompt before Claude responds
  • UserPromptSubmitstrips privacy tags, enqueues prompt to worker for storage
  • PostToolUsecaptures tool call results as observations
  • Summarystores the session summary generated by Claude
  • SessionEndtriggers learning extraction pipeline on session close
Worker Service
▼
  • worker-service.tsExpress entrypoint on :37777, managed by Bun
  • SessionManagertracks active sessions and their lifecycle state
  • SearchManagerorchestrates local + team search, injects SyncClient
  • SSEBroadcasterpushes real-time events to Viewer UI over SSE
  • GeminiAgent / SDKAgent / OpenRouterAgentpluggable LLM providers for AI processing tasks
  • ResponseProcessornormalizes and stores LLM agent output
  • SessionCompletionHandlerfinalizes session state on close
SQLite / Local DB
▼
  • DatabaseSQLite3 connection manager at ~/.claude-mem/claude-mem.db
  • SessionStoresession CRUD; injects items into SyncQueue after writes
  • Observationsstores per-turn observation records
  • Summariesstores session summaries
  • Promptsstores user prompts
  • Sessionssession metadata rows
  • SyncQueueoutbound sync items table (storage layer; behavior in Sync Pipeline)
  • MigrationRunnerversioned schema migrations, auto-runs on startup
Sync Pipeline
▼
  • SyncQueuebehavior layer — enqueue/dequeue outbound sync items
  • SyncClientHTTP client that pushes queue items to Vercel backend
  • SyncWorkertick-based worker; drains queue, triggers extraction on SessionEnd
  • LearningExtractorLLM distillation → {claim, evidence, scope, confidence} structs
  • ConflictDetectorLLM dedup on approval path; runs server-side via Vercel
Chroma / Vector Search
▼
  • ChromaSyncwrites observation embeddings to local Chroma store in parallel with SQLite
  • ChromaMcpManagerMCP interface exposing Chroma to external tools
  • HybridSearchStrategymerges SQLite full-text results with Chroma vector results
  • ChromaSearchStrategysemantic similarity search against the embedding store
Vercel API
▼
  • push.tsreceives sync payloads from agents
  • status.tsreturns queue depth and sync health
  • invalidate.tsmarks a learning as invalidated
  • search.tsunified search across observations + approved learnings
  • timeline.tstimeline queries against Supabase
  • agents/ (create, revoke, list)agent key management — bearer tokens for API auth
  • learnings/reviewapprove/reject/edit pending learnings from dashboard
  • health.ts / db-check.tsops endpoints for uptime and DB connectivity checks
Supabase
▼
  • observationssynced observation rows from all agents
  • sessionssynced session metadata
  • summariessynced session summaries
  • learningsextracted learnings with status: pending | approved | rejected
  • agent_keysbearer tokens for API authentication
Review Dashboard
▼
  • public/dashboard/DOM-safe HTML/JS UI served by Vercel
  • Authbearer token (agent key) — no Supabase Auth for POC
  • Approve actionruns ConflictDetector server-side, promotes learning to approved
  • Reject actionmarks learning as rejected, removes from search corpus
  • Edit actioninline claim editing before approve; buttons disabled during in-flight requests
Viewer UI
▼
  • React SPAserved at localhost:37777, built to plugin/ui/viewer.html
  • Feedpaginated stream of observations and summaries
  • ObservationCard / SummaryCardtyped entry renderers with icons and metadata
  • SSE streamreal-time updates pushed from SSEBroadcaster
  • SearchRouteslocal search UI backed by worker search API
  • ContextSettingsModalconfigure context injection parameters
